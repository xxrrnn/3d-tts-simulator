# Extreme Branch Analysis (Threshold = 4.0x)

## 📊 极端值定义

**Threshold = 4.0x** 是一个非常严格的标准，只捕获最极端的差异：

- **极端大 (Extreme LARGE)**: 某个branch的token数量 ≥ 4倍其他branches平均值
- **极端小 (Extreme SMALL)**: 某个branch的token数量 ≤ 其他branches平均值/4

**示例**：
- `[100, 20, 25]` → 极端大：100 (是20和25平均值的4.4倍) ✅
- `[1, 20, 25]` → 极端小：1 (是20和25平均值的1/22.5) ✅
- `[50, 20, 25]` → 无极端 (50是20和25平均值的2.2倍，不到4倍)

## 📈 极端情况的普遍性

**Threshold = 4.0x 下的出现频率**：

| 类型 | 出现率 | 说明 |
|------|--------|------|
| **极端大** | 20-35% | 只有约1/4-1/3的决策存在极端大 |
| **极端小** | 70-83% | **大部分决策都存在极端小！** |
| **同时存在** | 19-29% | 少数情况下同时存在两个极端 |

**为什么极端小这么常见？**
- 在beam search中，某些branch会"早停"（生成了终止符或达到停止条件）
- 这些branch只生成1-5个token就结束了
- 而其他branch可能继续生成20-50个token
- 因此极端小的情况（1 vs 20-50）非常常见

## 💡 实践意义

### 1. 预测模型行为

**场景**：某个问题的beam search中，一个branch生成了200个token，其他5个branch都生成了40-50个token

| 模型 | 预测 | 依据 |
|------|------|------|
| **Qwen2.5-Math-PRM** | 只有**12%**概率选择200token的branch | 极端大选择率接近随机 |
| **Skywork-PRM** | 有**40%**概率选择200token的branch | 极端大选择率是随机的2.3倍 |

**结论**：Skywork-PRM更可能产生长输出，Qwen2.5-Math-PRM更可能产生短输出

### 2. 优化Beam Search参数

**如果你观察到生成过长的问题**：
- ✅ 使用Qwen2.5-Math-PRM作为reward model
- ✅ 它会主动避免极端长的branch (只有13.6%选择率)
- ✅ 倾向选择极端短的branch (34.4%选择率)

**如果你需要详细的推理过程**：
- ✅ 使用Skywork-PRM作为reward model
- ✅ 它会偏好极端长的branch (39.7%选择率)
- ✅ 避免极端短的branch (只有11.9%选择率)

### 3. 理解"选择率不是100%"

即使在4倍差异下，选择率最高也只有52.5%，原因：

1. **质量优先于长度**：
   - 即使token少，如果生成质量高（reward高），仍可能被选
   - Beam search综合考虑长度和质量

2. **多样性保留**：
   - 不是所有问题都适合同一种策略
   - 简单问题适合短答案，复杂问题需要长推理

3. **随机性**：
   - Temperature等参数带来的随机性
   - 使模型不会完全僵化

## 🎯 总结

### 核心发现

1. **Threshold=4.0x 只捕获最极端的20-30%情况**
   - 极端大：20-35%的决策存在
   - 极端小：70-83%的决策存在

2. **模型偏好在极端情况下被放大**
   - Qwen2.5-Math-PRM: 极端小选择率达到34-52% (vs 整体MIN 31%)
   - Skywork-PRM: 极端大选择率达到36-40% (vs 整体MAX 27%)

3. **两个模型的策略完全相反**
   - Qwen2.5-Math-PRM偏好小的倍数: 2.5-4.6x
   - Skywork-PRM偏好大的倍数: 1.5-3.3x

4. **选择不是绝对的**
   - 最高选择率52.5%，说明还有47.5%选择其他
   - 模型综合考虑长度、质量、正确性等多个因素
