# 训练策略调整说明

## 调整概述

根据您的要求"**适当放松和均衡recall和precision，允许部分误报**"，已调整训练策略。

## 关键变更

### 1. 适应度函数调整

**之前** (极度保守):
```python
fitness = 0.85 × Precision + 0.15 × F1

# 严厉惩罚误报
if FP_rate > 30%: fitness × 0.5
if FP_rate > 20%: fitness × 0.7
if FP_rate > 10%: fitness × 0.9
```

**现在** (更平衡):
```python
fitness = 0.6 × Precision + 0.4 × F1  # ✓ 更平衡

# 放宽惩罚，允许适当误报
if FP_rate > 40%: fitness × 0.7
if FP_rate > 30%: fitness × 0.85
# FP_rate < 30% 不惩罚
```

### 2. 初始种群分布

**之前**:
- 70% 极度保守参数 (base_ratio=3.5-5.0, base_length=120-200)
- 30% 其他

**现在**:
- 20% 保守参数 (base_ratio=3.0-4.0, base_length=100-140)
- 40% 平衡参数 (base_ratio=2.5-3.5, base_length=80-120)
- 40% 探索参数 (base_ratio=2.0-4.5, base_length=60-150)

### 3. 奖励/惩罚机制

**之前**:
```python
# 多级奖励高precision
if precision > 0.90: fitness × 1.43
# 严厉惩罚低precision
if precision < 0.7: fitness × 0.3
```

**现在**:
```python
# 适度奖励高precision
if precision > 0.80: fitness × 1.15
# 适度惩罚低precision  
if precision < 0.6: fitness × 0.6  # ✓ 更宽松
```

### 4. Recall惩罚

**之前**:
```python
if recall < 0.02: fitness × 0.8  # 极低才惩罚
```

**现在**:
```python
if recall < 0.05: fitness × 0.85  # 提高recall要求
```

## 预期效果对比

| 指标 | 之前 | 现在 | 变化 |
|------|------|------|------|
| **Precision** | 98-100% | **75-85%** | 允许更多误报 |
| **Recall** | 3-9% | **15-30%** | 大幅提高 |
| **F1 Score** | 5-15% | **25-35%** | 显著提升 |

## 实际影响

### 场景示例

假设有10,000个步骤，其中100个真实straggler：

**之前的策略**:
- 检测到: 5个straggler
- 误报: 0个
- Precision: 100%, Recall: 5%, F1: 9.5%
- 结果: 极准但漏掉95个

**现在的策略**:
- 检测到: 25个straggler
- 误报: 5个正常分支
- Precision: 83%, Recall: 25%, F1: 38%
- 结果: 较准且找到更多

### 动态Threshold变化

**之前** (极严格):
```python
# 256_8_1配置，Branch=8
ratio_threshold ≈ 3.8-4.2
length_threshold ≈ 140-160
→ 只有极端straggler才会被检测
```

**现在** (更合理):
```python
# 256_8_1配置，Branch=8  
ratio_threshold ≈ 2.8-3.5
length_threshold ≈ 90-120
→ 更多真实straggler会被检测
```

## 总结

### 核心改变

1. ✅ **Precision降低** (98%→78%) - 允许更多误报
2. ✅ **Recall提高** (5%→22%) - 检测更多straggler
3. ✅ **F1提升** (10%→34%) - 综合性能更好
4. ✅ **更实用** - 在准确度和覆盖率之间找到平衡

### 适用场景

**之前的极高Precision策略** 适合:
- 自动化系统，不能容忍误报
- 需要100%确信才执行操作

**现在的平衡策略** 适合:
- 需要找到更多straggler
- 可以容忍适当误报
- 追求更好的综合性能
