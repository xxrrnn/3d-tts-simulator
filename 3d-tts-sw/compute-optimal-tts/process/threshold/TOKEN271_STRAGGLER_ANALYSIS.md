## 深入分析

### 为什么未选中的Straggler更多包含Token 271？

**假设1：Token 271的存在表明模型"想要"结束但未成功**
- 未选中的straggler可能是模型多次尝试用`'\n\n'`结束
- 但由于某些原因（如reward不高）继续生成
- 导致序列中出现多次271但概率都很低
- 最终这些branch因为质量问题未被选中

**假设2：选中的Straggler是"高质量的长推理"**
- 选中的straggler可能是**刻意的长推理链**
- 不需要频繁的step分隔符
- 是一个连贯的长推理过程
- 因此被选中

**假设3：Token 271在top-k中但概率低，说明它是"备选"不是"主选"**
- Token 271出现在top-k列表中
- 但概率接近0，排名很低
- 说明模型考虑过这个token但没有选择它
- 这些"考虑但未选择"的token积累导致未选中straggler有更多271

### Token 271占比的意义

| 类别 | Token 271平均占比 |
|------|------------------|
| 选中的Straggler | 5.2% |
| 未选中的Straggler | **12.3%** (2.4x) |

未选中的straggler中，平均12.3%的token位置将271作为候选，这个比例是：
- 选中straggler的2.4倍
- 非straggler的1.5倍

**这强烈暗示**：模型在生成未选中straggler时，频繁地"犹豫"是否要用`'\n\n'`结束。
